# SELECTION.md — What to include in daily memory

Use this before writing anything to `memory/YYYY-MM-DD.md` or `MEMORY.md`.

## The test

Ask: "If another agent read only this, what would it *do differently* tomorrow?"

Only include content that changes decisions and behavior. Not content that merely adds detail.

---

## Rules

1. **Pattern > episode** — Include repeatable dynamics (loops, triggers, default responses). Exclude one-off events unless they explain the pattern.

2. **Constraints > preferences** — Include "what makes this hard". Exclude narrative unless it clarifies a constraint.

3. **Levers > explanations** — Include actionable interventions (what to do / not do). Don't psychoanalyze motives.

4. **Commitments > feelings** — Include commitments that guide action. Feelings matter, but summarize them as triggers or signals, not as the main content.

5. **Portable phrasing** — Prefer short sample language that encodes the stance without overexplaining.

6. **Minimize sensitive detail** — If a detail is identifying or intimate and not necessary for the pattern, leave it out. If it *is* necessary, keep it abstract.

7. **No new claims** — Only include what is directly supported by what was said or done. Label inferences as inferences, or omit them.

---

## Practical cut test

Before writing a bullet, ask: if I removed this, would the layer become less useful for:
- avoiding a failure mode,
- choosing a better response in the moment,
- preserving the relationship goal?

If not — cut it.
